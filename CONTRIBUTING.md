# Contributing

Thanks for contributing to this fork.

## Scope

- Keep UX and behavior compatible with KRunner SpotifyEnhanced usage (`spe ...`).
- Prefer focused pull requests.
- Preserve credits to original upstream project.

## Before opening a PR

- Run `./install.sh` successfully.
- Run `bash -n install.sh debug.sh uninstall.sh`.
- If possible, run plugin via `./debug.sh` and test key commands.

## Commit style

- Use clear, imperative commit messages.
- Example: `fix: handle missing config keys gracefully`

## Coding rules

- Keep changes minimal and explicit.
- Avoid introducing unrelated refactors.
- Keep shell scripts POSIX-friendly where practical.

## Credits and licensing

- Do not remove upstream attribution.
- Keep GPL-3.0 compatibility intact.
